Filtros de Visualização (SDSF)

ZCTO & PutProc

TSO ZCTO: Controle de terminais. Usado para derrubar sessões travadas e limpar o caminho para o faturamento.

PUTPROC: Atualiza a lógica de um processo sem derrubar o sistema. Caminho: ZCTO -> 4E -> Histórico.

Sempre peça o estatístico (statistic) do dia anterior. No comando V, verifique o TOTAL.

Tabelas (Rins/TCO)

Tabelas de Controle de Rotinas (Rins)

As tabelas Rins são responsáveis pelo controle das rotinas de processamento no Mainframe. Elas definem os parâmetros de execução, dependências e horários.

  • Rin01: Controle de Bilhetagem - Define os jobs de processamento de CDRs.
  • Rin02: Controle de Faturamento - Gerencia as rotinas de fechamento mensal.
  • Rin03: Controle de Transmissão - Coordena envios para sistemas externos.

Comando de Verificação: Use TSO DISPLAY RINS para visualizar o status atual.

Tabelas de Controle Operacional (TCO)

As tabelas TCO controlam as operações críticas do sistema, incluindo failovers e recuperações.

  • TCO01: Controle de Ambientes - Define os IPs e configurações por ambiente (PRD2, PRD3).
  • TCO02: Controle de SLA - Monitora os tempos de resposta e alertas.
  • TCO03: Controle de Espaço - Gerencia alocação de disco e limpeza.

Atualização: Sempre verificar as tabelas antes de manutenções críticas.

As 4 Planilhas de Rotina

Plan 01: Bilhetadoras

Conferir no e-mail; anotar o último horário que consta no corpo do e-mail recebido.

Plan 02: PutProc (TSO)

Pegar informações via TSO ZCTO -> 4E. Verificar o TOTAL no modo View (V).

Plan 03: Espaço em Produção

AS400: Checa e-mail de marcação de disco (SP e Rio). Copiar valor.

Mainframe: Usar comando STG LAST para pegar o espaço de PRD2 e PRD3.

Plan 04: SLA (ODATE D-1)

Caminho: SD (DA/JOB) -> Pre -> Owner IBMPROD. Comando I mostra a fila para entrar.

Grafana Dashboards

Dashboard Principal

Visão Geral do Sistema

Monitora CPU, memória e jobs ativos em tempo real. URL: http://grafana.3am.com/d/main

Métricas Chave: Utilização PRD2/PRD3, fila de jobs, alertas SLA.

Dashboard de SLA

Monitoramento de Performance

Acompanha os tempos de processamento das rotinas críticas. Inclui gráficos de tendência histórica.

Alertas: Notificações automáticas quando SLA é violado.

Dashboard de Infraestrutura

Recursos de Sistema

Monitora espaço em disco, rede e conectividade entre ambientes.

Comandos: Integrado com comandos STG e ZCTO para alertas proativos.


Acesso: Use credenciais do domínio 3AM. Dashboards atualizam a cada 5 minutos.

Relatórios: Exportar dados para as planilhas diárias via API do Grafana.

Gestão Jira (PDST & OPS)

PDST: Execuções (Rename, Transmissão, Sysout). Se houver ABEND, escrever no campo de texto antes de atualizar.

OPS: Local dos Abends. Localizar via FIND [StepName]. Usar F11 (Direita) e F10 (Esquerda).

Fluxo OPS: CTMS Tela 3 -> Find Job -> Se Hold (H) -> Comando O e Y (Force OK) -> Comando F (Free) -> Printar evidência.
